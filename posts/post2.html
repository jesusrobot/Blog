Git y Github no es la misma cosa, que eso quede claro desde el principio. Git es un sistema de control de versiones (ve más de git aquí) mientras que github es una plataforma de desarrollo colaborativo de software para alojar proyectos.

Github nos permite hacer muchas cosas geniales, aquí algunas:

  • Crear páginas web html, css y js para nuestros proyectos.
  • Push, pull y clonaciones ilimitadas.
  • ssues, wikis y estadísticas.
  • Crear organizaciones.
  • Repositorios públicos y privados ilimitados.

Entre otras muchas cosas que están en su plan gratuito, por lo que no es excusa el precio para comenzar a aprenderlo.

Ya para empezar a meternos en materia también hay que aclarar que github no es la única plataforma de alojamiento de repositorios, hay muchas más como por ejemplo gitlab y bitbucket. Pero si es la más popular, y por mucho, por lo que probablemente sea la mejor opción si estás empezando con esto de los sistemas de gestión de versiones.

Asi que ya aclaradas algunas cosas de github y los beneficios de esta plataforma vamos a lo importante.

Crear una cuenta de github

Para comenzar hay que crear una cuenta de github para eso vamos a www.github.com.

Pagina principal de github.com

Una vez dentro hay que llenar el formulario de registro que se muestra al inicio y presionamos sign up for github, Luego github nos pedirá que resolvamos un captcha muy divertido y que seleccionemos un plan.

Github tiene dos planes individuales y 3 para empresas. Por el momento escojamos el plan gratis, ya despues si lo necesitamos podemos recurrir a cualquiera de los otros planes de paga.

Ya para terminar git nos pedirá que contestemos una pequeña encuesta la cual podemos omitir si queremos y que verifiquemos nuestro correo electrónico.

Y listo ya tenemos una cuenta de github.

Crear un repo en github

Para crear nuestro primer repositorio en github primero hay que ubicar en la parte superior derecha un botón con el icono de + símbolo universal de agregar XD. Y de las opciones que se nos despliega elegimos new repository.

Feed de github

Después hay que darle nombre a nuestro repositorio, y si queremos podemos escribir una breve descripción sobre lo que trata el repo que estamos creando. También podemos decidir si va a ser público o privado (¡¡¡gratis!!!).

Esto ya lo configuras como quieras pero para poder seguir con esta guía de lo mejor manera hay que tener desactivada la opción de inicializar un archivo README.md al crear el repositorio.

Formulario para crear un repo de github

Y terminamos de crear nuestro repo dando click en el botón Create repository

Como pushear nuestros archivos (git push)

Cuando usamos git push lo que estamos haciendo es actualizar las referencias de cambios del repositorio remoto con las del repositorio local.

Para hacer git push primero hay que agregar el origen al cual vamos a mandar estas referencias, para eso usamos el siguiente comando:

git remote add origin url
